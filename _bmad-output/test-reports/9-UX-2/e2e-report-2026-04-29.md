## Bugs Found & Fixed

### 🐛 BUG-1 (P0, FIXED) — Scenario Simulator squeezes report content at 1440px

**File:** [crypto-report-layout.tsx](../../../../nowing_web/components/new-chat/report/crypto-report-layout.tsx)  
**Root cause:** `xl` (1280px) breakpoint too small — chat content area (~688px after left nav + docs panel) can't fit 3 columns: TOC(192px) + content(min 300px) + Simulator(300px) = 792px minimum.  
**Fix applied:** Changed `xl:hidden` → `2xl:hidden` and `hidden xl:block` → `hidden 2xl:block`. Simulator now stacks below on screens <1536px, goes sticky-right only on ≥1536px.

```diff
- <div className="mt-6 xl:hidden">
+ <div className="mt-6 2xl:hidden">
...
- <div className="hidden xl:block">
+ <div className="hidden 2xl:block">
```

### 🐛 BUG-2 (P2, FIXED) — HTML comments rendered as visible text

**File:** [markdown-text.tsx](../../../../nowing_web/components/assistant-ui/markdown-text.tsx)  
**Root cause:** LLM emits `<!--follow-ups:[...]-->` trailing comments in synthesis output. remark-gfm does not strip HTML comments — they appear as literal text in DOM.  
**Fix applied:** Added strip regex in `preprocessMarkdown()` before markdown pipeline:

```ts
// Strip HTML comments emitted by LLM (e.g. <!--follow-ups:[...]-->) — not valid markdown
content = content.replace(/<!--[\s\S]*?-->/g, "");
```
